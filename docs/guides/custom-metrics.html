Custom Metrics

Extend the evaluation capabilities by adding your own metrics.

🚧 Work in Progress

The current version of OpenVals (v0.0.2) requires minimal manual integration to register custom metrics into the core loop. This will be simplified in v0.3.

Defining a Metric

A metric in OpenVals is a simple function that takes the model's output and the expected output, and returns a numerical score (usually between 0 and 1).

python 
def word_count_similarity(pred: str, expected: str) -> float:
    p_count = len(pred.split())
    e_count = len(expected.split())
    
    if e_count == 0: return 0.0
    return 1.0 - min(abs(p_count - e_count) / e_count, 1.0)

Integrating Custom Metrics

To use your custom metric, you currently need to modify the Evaluator instance or pass it during initialization if supported by your version.

Example Flow

  1. Define your function.
  2. Import it into your evaluation script.
  3. Assign it to the evaluation logic.
💡 Roadmap Note

In v0.3, you will be able to pass a list of custom metric functions directly to the Evaluator constructor.
