Custom Models

How to create model adapters for any AI system or LLM provider.

Overview

OpenVals is model-agnostic. To evaluate a new model, you simply need to create a class that implements the BaseModel interface. This class acts as a bridge between OpenVals and your model's API (OpenAI, Anthropic, Ollama, etc.).

Step-by-Step Implementation

1. Subclass BaseModel

Import BaseModel and create your class.

from openvals.models.base import BaseModel

class MyOpenAIModel(BaseModel):
    def __init__(self, api_key):
        self.api_key = api_key

2. Implement generate()

The generate method takes a prompt string and should return the model's response string.

    def generate(self, prompt: str) -> str:
        # Code to call your model API
        response = call_openai_api(prompt, self.api_key)
        return response.text

Full Example: OpenAI Adapter

python 
import openai
from openvals.models.base import BaseModel

class OpenAIModel(BaseModel):
    def __init__(self, model_name="gpt-3.5-turbo"):
        self.client = openai.OpenAI()
        self.model_name = model_name

    def generate(self, prompt: str) -> str:
        completion = self.client.chat.completions.create(
            model=self.model_name,
            messages=[{"role": "user", "content": prompt}]
        )
        return completion.choices[0].message.content

Using Your Custom Model

Once implemented, you can pass an instance of your class directly to the Evaluator.

model = OpenAIModel(model_name="gpt-4")
evaluator = Evaluator(model, dataset)
results = evaluator.run()
