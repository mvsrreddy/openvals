🛡️ Core Capabilities

0. Decision Reliability Score (DRS)

A production-oriented scoring framework designed to evaluate whether an AI model can be trusted in real-world deployment environments beyond just accuracy.

1. Model Evaluation

Evaluate model outputs against structured datasets using accuracy, embedding-based semantic similarity, reliability, safety, and consistency.

2. Multi-Model Benchmarking

Compare multiple models under the same conditions with side-by-side evaluation, normalized scoring, and automated model ranking.

3. Scoring Engine

Weighted scoring aligned to business priorities. Customize weights per use case to balance accuracy, cost, and latency.

4. Extensible Architecture

Plug-and-play model adapters, custom metrics support, and scalable evaluation pipelines designed for enterprise growth.

5. Recommendation Engine

Suggests the best model for your dataset using tradeoff-aware ranking (speed vs accuracy vs safety) and use-case based selection.

6. Semantic Intelligence Engine

Embedding-powered evaluation using all-MiniLM-L6-v2 for meaning-aware evaluation and reduced keyword bias.

🚀 Why OpenVals?

AI models are powerful—but without proper validation, they are unpredictable, insecure, and hard to trust. Most AI evaluation tools stop at basic metrics. OpenVals exists to solve that.

It provides a structured way to quantify trust, risk, and performance, ensuring your models are deployment-ready.

Quick Example

python 
from openvals.core.evaluator import Evaluator
from openvals.models.ollama_model import OllamaModel
from openvals.datasets.loader import load_dataset

# Load dataset and local model
dataset = load_dataset("examples/sample_eval.json")
model = OllamaModel("llama3")

# Run evaluation with Trust Scoring
evaluator = Evaluator(model, dataset)
result = evaluator.run()

print(f"Overall Trust Score: {result['overall_score']}")
print(f"DRS Score: {result['drs_score']}")
