OpenVals Documentation Structure
==============================

This directory contains the source for the OpenVals documentation website, a static, minimalist, and premium documentation site built with pure HTML, CSS, and JS.

Root Files:
-----------
- index.html: The landing page featuring the value proposition, core mission, and a feature grid.
- getting-started.html: Installation instructions, dependencies (sentence-transformers), and "first evaluation" guide.
- core-concepts.html: Deep dive into the architecture, the "Trust Score" philosophy, and the "Decision Reliability Score" (DRS).
- examples.html: Curated code samples for Python API and CLI workflows.
- roadmap.html: Project milestones from v0.1.5 through v1.0.0.

API Reference (/api):
---------------------
- evaluator.html: Documentation for the core Evaluator class, including task-specific logic (classification vs generation).
- models.html: Guide to Model Adapters (BaseModel, DummyModel, and OllamaModel).
- datasets.html: Details on the built-in dataset registry and JSON schema for custom data.
- metrics.html: Comprehensive reference for all 10+ metrics (Accuracy, Semantic, Reliability, Safety, Consistency, etc.) including an interpretation guide.
- benchmarking.html: Explanation of the BenchmarkRunner and mathematical normalization logic (Logarithmic Latency scaling).
- scoring.html: Details on weighted scoring calculations and DRS integration.
- recommendation.html: API for the AI Advisor / Recommendation Engine, including risk thresholds and tradeoff analysis.
- reporting.html: Documentation for the automated HTML report generator.
- safety.html: Detailed guide to the Safety Engine layers (Jailbreak, Toxicity, and Unsafe Keywords).

Guides (/guides):
-----------------
- configuration.html: How to use YAML presets (Finance, Legal, Healthcare) and custom weighting.
- cli.html: Documentation for the Typer-based CLI tool (benchmark, datasets, version commands).
- custom-model.html: Step-by-step tutorial on building custom model adapters.
- custom-metrics.html: Instructions for extending the framework with user-defined evaluation metrics.

Assets & Logic:
---------------
- css/styles.css: The central design system (Monochrome Dark/Light themes, glassmorphism, and responsive layouts).
- js/app.js: Core UI logic handling sidebar transitions, theme toggling, and interactive components.
- js/search-index.js: A searchable registry of all documentation content for the global search bar.
- assets/: Contains the project logo and branding assets.

Summary for Pull Request:
-------------------------
The documentation has been synchronized with OpenVals v0.1.5. It provides 100% coverage of the new Safety Engine, Configuration Presets, and Typer CLI. The site follows a premium, minimalist aesthetic designed to "WOW" the user while providing deep technical transparency into the library's mathematical and structural implementation.
